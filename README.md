## Metodologia Pedagogica

### 1. Metodo Feynman
> *"Si no puedes explicarlo de forma simple, no lo entiendes realmente"* - Richard Feynman

Cada concepto se explica:
- Como a un nino de 5 anos
- Con analogias del mundo real
- Sin jerga innecesaria

### 2. Repeticion Espaciada (SM-2)
Sistema cientifico de repaso que maximiza retencion:
```
Dia 1 → Dia 3 → Dia 7 → Dia 14 → Dia 30 → Dia 60
```
Recordaras cada algoritmo incluso meses despues!

### 3. Practica Deliberada
- Ejercicios progresivos por dificultad
- Feedback inmediato con tests automaticos
- Pistas socraticas que guian sin dar la respuesta

## Arquitectura

La aplicación sigue una arquitectura **cliente-servidor** con separación clara entre frontend y backend:

```
┌─────────────────────────────────────────────────────────────────┐
│                         FRONTEND                                 │
│  React + TypeScript + Vite + Tailwind CSS                       │
│  ┌─────────────┬─────────────┬─────────────┬─────────────┐     │
│  │   Pages     │  Components │   Context   │    Types    │     │
│  │  (Routing)  │    (UI)     │  (State)    │  (Models)   │     │
│  └─────────────┴─────────────┴─────────────┴─────────────┘     │
└────────────────────────────┬────────────────────────────────────┘
                             │ HTTP/REST API
                             ▼
┌─────────────────────────────────────────────────────────────────┐
│                         BACKEND                                  │
│  FastAPI + Python + Pydantic                                    │
│  ┌─────────────┬─────────────┬─────────────┬─────────────┐     │
│  │   Routers   │   Models    │   Sandbox   │    Data     │     │
│  │ (Endpoints) │  (Schemas)  │  (Secure)   │  (In-mem)   │     │
│  └─────────────┴─────────────┴─────────────┴─────────────┘     │
└─────────────────────────────────────────────────────────────────┘
```

## Instalacion y Ejecucion

### Requisitos Previos
- **Python 3.10+**
- **Node.js 18+**
- **npm** o **yarn**

### Backend

```bash
# Navegar al directorio backend
cd backend

# Crear entorno virtual (opcional pero recomendado)
python -m venv venv
source venv/bin/activate  # Linux/Mac
venv\Scripts\activate     # Windows

# Instalar dependencias
pip install -r requirements.txt

# Ejecutar servidor de desarrollo
uvicorn main:app --reload --port 8000
```

La API estará disponible en: `http://localhost:8000`
- Documentación Swagger: `http://localhost:8000/docs`
- Documentación ReDoc: `http://localhost:8000/redoc`

### Frontend

```bash
# Navegar al directorio frontend
cd frontend

# Instalar dependencias
npm install

# Ejecutar servidor de desarrollo
npm run dev
```

La aplicacion estara disponible en: `http://localhost:3000`

## Seguridad en Ejecucion de Codigo

El backend implementa un **sandbox seguro** para ejecutar codigo Python del usuario:

- Lista blanca de funciones built-in permitidas
- Imports seguros limitados (collections, heapq, itertools, functools)
- Bloqueo de operaciones peligrosas:
  - `import os`, `import sys`, `import subprocess`
  - `eval()`, `exec()`, `open()`, `input()`
  - Acceso a `__builtins__`, `__globals__`, `__class__`
- Timeout de ejecucion de 5 segundos

## Sistema de Progreso y Retencion

### Estado del Usuario (React Context)
- **Lecciones**: No iniciada → En progreso → Completada
- **Ejercicios**: No iniciado → En progreso → Completado/Fallido
- **Modulos**: Bloqueado → Disponible → Completado

### Sistema de Repeticion Espaciada (SM-2)

Basado en el algoritmo SuperMemo 2 para maximizar retención:

```
┌──────────────────────────────────────────────────────────┐
│  CALIDAD DE RESPUESTA                                    │
├──────────────────────────────────────────────────────────┤
│  5 - Perfecto       → Intervalo × 2.5                    │
│  4 - Fácil          → Intervalo × 2.0                    │
│  3 - Bien           → Intervalo × 1.5                    │
│  2 - Difícil        → Intervalo × 1.0 (repetir)          │
│  1 - Muy difícil    → Reiniciar a 1 día                  │
│  0 - Olvidé         → Reiniciar a 1 día                  │
└──────────────────────────────────────────────────────────┘
```

**Progresión típica**: 1 día → 3 días → 7 días → 14 días → 30 días → 60 días

### Estadisticas Rastreadas
- Total de lecciones completadas
- Total de ejercicios resueltos
- Tiempo total dedicado
- Racha actual y mas larga
- Ejercicios por dificultad (Easy/Medium/Hard)
- Items dominados vs en aprendizaje
